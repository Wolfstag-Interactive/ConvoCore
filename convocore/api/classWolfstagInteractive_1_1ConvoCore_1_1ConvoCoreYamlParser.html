Detailed Description

Provides methods to parse YAML text into a conversation data structure and manage associated dialogue configurations.

Parsing runs as a three-phase pipeline:

  1. Pre-flight — ConvoCoreYamlPreprocessor scans the raw text for file-level problems (BOM, merge conflicts, multiple documents, tab indentation, smart quotes, zero-width characters).
  2. State machine transform — ConvoCoreYamlPreprocessor walks the file and applies the escape pipeline only to locale value lines inside LocalizedDialogue blocks; everything else passes through unchanged.
  3. Post-process validation — after YamlDotNet deserialises the processed YAML, ValidateRoundTrip checks for silent data corruption (embedded control characters that indicate a misinterpreted escape sequence).

Parses yamlText into a Dictionary<conversationKey, List<DialogueYamlConfig>>.

Locale keys are normalised to lower-invariant and stored in a case-insensitive dictionary.

Exceptions
InvalidOperationExceptionThrown when pre-flight or post-process validation finds blocking errors. The exception message contains the full human-readable diagnostic block.
YamlExceptionRe-thrown from YamlDotNet when the preprocessed YAML is syntactically invalid.
