Detailed Description

Resolves, tracks, and releases prefab-based character displays for a single UI instance.

Resolution order for each character:

  1. Scene registry lookup by characterId — if a registrant is found the scene instance is used. ConvoCore never spawns, pools, or destroys scene-resident characters.
  2. Prefab spawn — the configuration entry's CharacterPrefab is drawn from ConvoCorePrefabPool and returned to it on release.

A console message is emitted for whichever path is taken so behavior is transparent without requiring manual source-mode declarations on the representation asset.

Active entries are tracked by the Transform passed as the slot anchor. Passing the same Transform again releases the previous occupant before placing a new one. Scene-resident characters are not tracked by slot and are never pooled or destroyed here.
