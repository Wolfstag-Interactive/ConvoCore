Detailed Description

Character behaviour type that wraps a list of other behaviour types and selects between them based on a conversation index counter or an optional named condition.

Each time OnConversationBegin is called the active behaviour is selected from the list using the current index (which increments automatically). Optionally a ConditionKey and ISequencedPresenceCondition can override the index-based selection.

All lifecycle calls (OnConversationBegin, OnConversationEnd) are forwarded only to the currently active sub-behaviour.

Use case: the same scene has multiple dialogue modes and requires different character placement strategies depending on which conversation is active.
