Detailed Description

Abstract ScriptableObject base for all presence types.

A presence determines how characters are placed and managed in 3D world-space conversations. It is responsible for resolving a live IConvoCoreCharacterDisplay for each character that appears on a line, and for cleaning up at conversation end.

The spawner is passed into ResolvePresence at call time. The presence is a ScriptableObject and must not hold serialized references to scene objects. Runtime-only scene references may be cached in [System.NonSerialized] fields.

Resolve the live IConvoCoreCharacterDisplay for the given representation.

Called once per character per dialogue line by ConvoCoreSampleUI3D.

Implementations that cache displays across lines (e.g. WorldPointPresence) should return the cached instance on subsequent calls for the same character rather than requesting a new spawn from the spawner.

Return null if the character cannot or should not be resolved (e.g. ExternalPresence for a spawn-from-prefab character). The 3D UI will skip expression application when null is returned.
